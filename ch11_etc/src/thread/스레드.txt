- Thread - 
프로세스 : 현재 실행중인 프로그램 
쓰레드 : 하나의 프로세스 안에 하나 이상의 Thread 사용
  ㄴ 실제 프로그램이 수행되는 최소 작업 단위

멀티 쓰레드
- Thread 클래스 사용
- Runnable  인터페이스 사용

실행대기상태에 있는 쓰레드를 스케쥴링에 의해 번간아 가면서 실행
cpu 싱글코어 : 스레드 우선순위에 따라 우선순위가 높은 
               스레드가 실행할 기회를 더 많이 받음
    쿼드코어 : 한번에 4개 스레드가 같이 실행

스레드할때 같은곳에서 돌려버리면 값이 계속 교체가 된다 (먼저했던 값이 나중에 했던 값으로)
ex) t1 = 100 , t2 = 200 이면 200이 나옴


동기화 메소드와 동기화 블록
- 멀티스레드 프로그램에서는 스레드들이 객체를 공유해서 작업하는 경우가 있음
- 공유해서 사용하는 경우 한 스레드가 먼저 사용중이라면 다른 스레드의 접근을 막아야 함
그래서 임계영역을 지정 => synchronized
